## Overview

This repository is a focused Python learning workspace for Class 12 SEM 3 practice. It contains short, interactive scripts that demonstrate:

- Nested loop based pattern printing
- Step-by-step logic building for beginner Python
- A factorial program with a simple interactive menu
- Quick access to both oneliner and normal versions of each solution

Each script is designed to be easy to run, easy to study, and easy to compare against alternate implementations.

## What You’ll Find

| Folder | Files | Purpose |
| --- | --- | --- |
| `Nested loop/` | `pattern-1.py` to `pattern-9.py` | Practice scripts for star and number patterns using nested loops |
| `Series/` | `factorial.py` | Interactive factorial example with browser-link options |

## Project Highlights

- Beginner-friendly Python examples with minimal setup
- Clear separation between pattern practice and series calculations
- Interactive prompts that let you choose between:
  - `one` for the oneliner version
  - `code` for the normal version
  - `run` to execute the logic locally
- Great for classroom revision, self-study, and quick concept reinforcement

## How To Run

1. Open the repository in VS Code or any Python-friendly editor.
2. Navigate to the file you want to study.
3. Run it with Python, for example:

   ```bash
   python "Series/factorial.py"
   ```

4. When prompted, choose one of the available options:
   - `one` to open the oneliner GitHub version
   - `code` to open the normal GitHub version
   - `run` to execute the example in the terminal

## Learning Focus

This project helps you practice:

- `for` loops
- Nested iteration
- Pattern formation logic
- Input handling
- Basic function structure
- Integer validation and simple error handling

## Suggested Study Flow

If you are learning from this repository, the best way to use it is:

1. Start with the earlier pattern files.
2. Observe how the loop structure changes from file to file.
3. Compare the oneliner and normal versions.
4. Run the scripts locally and predict the output before execution.
5. Move on to `factorial.py` for a simple series-based example.

## Why This Repo Is Useful

This is not a large framework project. It is intentionally compact, which makes it ideal for rapid concept revision. The value is in repetition, comparison, and understanding how a small change in loop boundaries changes the output completely.

## Notes

- The `run` option executes the local Python logic directly in the terminal.
